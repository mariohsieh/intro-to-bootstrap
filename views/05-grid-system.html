Grid System

Goals

  • Containers, rows, and columns

  • Column offsets

Overview

This lesson will cover one of Bootstrap's most powerful features, the grid system. First we will go over the container class and then move onto page layouts using rows and columns. From there, we'll breakdown the 12-column grid system and how to center, offset or re-order the columns.

Steps

Step 1

Containers

Bootstrap offers two types of container classes: container for centered fixed-width content, or container-fluid for full-width content.  Look at the two images below for comparison.  The red area is where your content would appear. 


<body class="container">
  ...
</body>

<body class="container-fluid">
  ...
</body>

If you set an HTML element with a container class, all the nested content inside will be confined to the horizontal boundaries set by the container.

This is useful if you want all your content to be uniformly-centered throughout your web application.


Rows and Columns

Page layout is handled by defining an HTML element (can use div but more semantic HTML5 tags will work as well) of class row and nesting a column(s) underneath as a direct child(ren). 


<section class="row">
  column
  column
  column
</section>

NOTE: not actually how to write columns, just placeholder text

Columns are created based on the concept that there are 12 units across the horizontal space in a row and any one column can have a width between 1 and 12 units.  The horizontal space of the row is defined by the width allowed from its parent HTML element.

If we were to create rows nested under the above examples of container and container-fluid, you can see the difference in horizontal space allowed.  Thus, the width of 1-unit for a column nested under container will be shorter than one under container-fluid, given all else is equal.

If there are multiple columns in a row, they will stack from left to right.  Any column that exceeds the maximum 12 allowable units will wrap onto the next line.

To create a column, we define an HTML tag with a class of col-XX-Y where XX is the screen size and Y is the number of units for the width of the column (1 to 12).

Here is the breakdown for the different screen sizes Bootstrap uses:

Screen Size Code to write Screen Width
Extra Small xs less than 768px
Small sm 768px and up
Medium md 992px and up
Large lg 1200px and up

The columns have also been styled with a background-color and given a fixed height to see the column dimensions better.  Also, assume these are all nested inside a body with container class)

Now looking at the profile page created in the Front End Curriculum, you can re-organize the page layout using Bootstrap: 


<body class="container">
  <header class="row">
    <article class="col-md-12">
      <!-- header content -->
    </article>
  </header>
  <section class="row">
    <article class="col-md-4">
      <!-- left column content -->
    </article>
    <article class="col-md-8">
      <!-- right column content -->
    </article>
  </section>
</body>

Step 2

Column Offsets

So far we have only created page layouts that have content starting from the left.  What if we want a centered single column 6-units wide?  Or how about a single 3-units wide column on the right side of the page?  This is where column offsets come in.

Offsets allow columns to move to the right so the first column isn't always positioned starting from the left. We can offset columns by adding an additional class col-XX-offset-Y. The 'XX' and 'Y' here also refer to screen size and number of units respectively.

Let's look at this example log-in screen: 


<section class="row">
  <article class=""col-md-4">
    <form>
      ...
    </form>
  </article>
</section>

As we can see, the content is nested inside a single column 4-untis wide that displays on the left. If we were to center this 4-unit column, we can add an offset of 4 units so the column occupies the 5,6,7,8 units in a 12-unit horizontal space. Therefore, it wil be centered. 


<section class="row">
  <article class=""col-md-4 col-md-offset-4">
    <form>
      ...
    </form>
  </article>
</section>

Step 3

Now on your index page, practice creating different page layouts using the grid system.  It might help to put some content inside the columns so you have something to see.  Or you can also explicitly set the height and background-color to see the column dimensions.

Here are some suggestions for page layouts to try:

  • One centered column of 2,4,6 and 8-units in width
  • A column 3-units wide on the right hand side
  • 4 columns of equal widths
  • 3 columns of different widths
